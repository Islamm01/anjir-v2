# Anjir Platform — Руководство по запуску

## Что это

Anjir — цифровая платформа доставки для Таджикистана.
Два сервиса: (1) Фрукты и сухофрукты, (2) Отправить посылку.
Полностью веб-интерфейсы — без Telegram, WhatsApp и внешних инструментов.

---

## Быстрый старт (локально)

### 1. Требования
- Node.js v18+
- PostgreSQL 14+

### 2. Установка

```bash
# Создать базу данных
createdb anjir

# Установить зависимости
npm install

# Скопировать и заполнить .env
cp .env.example .env
# Отредактируйте .env — заполните DATABASE_URL и JWT_SECRET

# Создать таблицы и загрузить данные
npx prisma migrate dev --name init
npm run db:seed

# Запустить
npm run dev
```

Открыть: http://localhost:3000

## Роли и доступ

| Роль | Назначается |
|------|-------------|
| `CUSTOMER` | По умолчанию при регистрации |
| `STORE_OWNER` | Вручную через Prisma Studio |
| `COURIER` | Вручную через Prisma Studio |
| `ADMIN` | Вручную через SQL |
| `OPERATOR` | Вручную через SQL |

Назначить роль через SQL:
```sql
UPDATE "User" SET role = 'ADMIN' WHERE phone = '+992900000000';
UPDATE "User" SET role = 'STORE_OWNER' WHERE phone = '+992901000001';
UPDATE "User" SET role = 'COURIER' WHERE phone = '+992902000001';
```

Или через Prisma Studio:
```bash
npx prisma studio
```

## Поток заказа (Marketplace)

```
Покупатель → Корзина → Оформление (адрес + оплата)
  ↓
Создаётся заказ (NEW_ORDER)
Уведомление → Магазин
  ↓
Магазин видит заказ в панели → Нажимает "Принять"
(STORE_CONFIRMED) → Уведомление → Покупатель
  ↓
Магазин нажимает "Начать сборку" (PREPARING)
  ↓
Магазин нажимает "Готов к выдаче" (READY_FOR_PICKUP)
Уведомление → Все онлайн-курьеры
  ↓
Первый курьер нажимает "Принять доставку" (COURIER_ASSIGNED)
Уведомление → Покупатель (имя курьера)
  ↓
Курьер забирает у магазина → "Забрал" (PICKED_UP)
Уведомление → Покупатель
  ↓
Курьер доставляет → "Доставил" (DELIVERED)
Уведомление → Покупатель
```

## Оплата

| Метод | Логика |
|-------|--------|
| Наличные | Курьер собирает при доставке. Автоматически помечается как завершено. |
| QR | Курьер показывает QR. Оператор подтверждает вручную в панели. |
| Перевод | Покупатель переводит и вводит референс. Оператор подтверждает. |

## Деплой (Vercel + Neon)

```bash
# 1. Создать базу на neon.tech
# 2. Скопировать connection string → DATABASE_URL в Vercel

# 3. Переменные окружения в Vercel:
DATABASE_URL=postgresql://...neon.tech/anjir?sslmode=require
JWT_SECRET=ваш-секретный-ключ-32-символа-минимум
NEXT_PUBLIC_APP_URL=https://ваш-домен.vercel.app
NODE_ENV=production

# 4. Задеплоить
vercel deploy

# 5. После деплоя — мигрировать БД
npx prisma migrate deploy

# 6. Загрузить данные
npm run db:seed
```

## SMS для OTP (продакшн)

В `lib/auth.ts` в функции `sendSms` раскомментируйте код для Eskiz.uz:
1. Зарегистрируйтесь на notify.eskiz.uz
2. Получите токен
3. Добавьте `ESKIZ_TOKEN` в переменные окружения
4. Раскомментируйте блок fetch в `sendSms`
